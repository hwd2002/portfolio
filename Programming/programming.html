My Programming Projects:

    I most often use lower-level systems or simulations of such systems to program my small games or simulators. I choose to do this because I want to understand my code and the underlying interactions that occur, rather than relying on a pre-defined physics system for example. Because I mostly write programs for my own self-improvement, most things I program are purposefully chosen to teach myself how to do new things, rather than produce a sellable product.

    Lately, I have been writing code using a "fantasy console" called Pico-8. The system simulates limited CPU, RAM, and disk space so writing efficient code under limitations is a challenge I enjoy. I also use Python3, C#, and Java for some projects that require higher processing power or capabilities.

    I wrote a small neural network that showed interesting results. This actual "game" the AIs played was very basic, but it did show improvement over time and the capability of 'learning' how to use different basic strategies.

    The original program was written in Java, but the code was lost on my old computer. Playable on this website is a recreation of this system in Pico-8's Lua implementation. This current implementation uses 7 input nodes, 4 middle nodes, and 3 output nodes. The neural net doesn't actively learn, instead using an "evolution" style of progression where each iteration of a network has slight changes from its parent.
